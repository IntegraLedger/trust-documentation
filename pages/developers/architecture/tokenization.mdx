# Tokenization Layer

## Overview

The tokenization layer binds ERC token standards to real-world contracts, enabling ownership transfer, fractional rights, and secondary markets for contract positions.

## Token Standards

### ERC-721: Unique Contracts

One-of-a-kind agreements:
- Employment contracts
- Custom service agreements
- Unique NDAs

```solidity
// Mint unique contract token
function mintContract(address to, bytes32 documentHash) external returns (uint256) {
    uint256 tokenId = _nextTokenId++;
    _mint(to, tokenId);
    documentToToken[documentHash] = tokenId;
    return tokenId;
}
```

### ERC-1155: Multi-Edition Contracts

Template agreements with multiple instances:
- Subscription services
- Standard license agreements
- Templated NDAs

```solidity
// Mint multi-edition contract
function mintBatch(address to, uint256[] ids, uint256[] amounts) external {
    _mintBatch(to, ids, amounts, "");
}
```

## Use Cases

### Transfer Contract Rights

```typescript
// Transfer employment contract to new employer
await contractNFT.transferFrom(oldEmployer, newEmployer, tokenId);
```

### Fractional Ownership

```typescript
// Split revenue-sharing contract
await erc1155.safeTransferFrom(owner, investor1, contractId, 25, ""); // 25%
await erc1155.safeTransferFrom(owner, investor2, contractId, 75, ""); // 75%
```

### Secondary Markets

```typescript
// List contract position for sale
await marketplace.list(contractNFT.address, tokenId, price);
```
