# Privacy Principles

## Overview

Privacy is not optional in Integra - it's a core architectural principle. Unlike systems that expose all data publicly, Integra uses **selective disclosure**, **off-chain storage**, and **access control** to protect sensitive information while maintaining verifiability.

## Core Privacy Requirements

### 1. Selective Disclosure

Share only what's necessary, nothing more.

**Challenge:** Prove you have a credential without revealing the credential.

**Solution:** Zero-knowledge proofs and attestations

**Example:**
```typescript
// Prove employment without revealing salary
const proof = {
  employer: "company-x.eth",  // Revealed
  duration: "2 years",        // Revealed
  salary: HIDDEN,             // Not revealed
  title: HIDDEN,              // Not revealed
  reviews: HIDDEN             // Not revealed
};
```

### 2. Off-Chain Storage

Never store PII (Personally Identifiable Information) on public blockchains.

**What Goes On-Chain:**
- Document hashes
- Attestation commitments
- Public keys
- Timestamps

**What Stays Off-Chain:**
- Document contents
- Personal details
- Sensitive business information
- Proprietary data

### 3. Access Control

Cryptographically enforce who can access what.

**Mechanisms:**
- Encryption keys per document
- Att

estation-based permissions
- Time-limited access grants
- Revocable credentials

### 4. Unlinkability

Prevent correlation attacks across contexts.

**Techniques:**
- Separate identities for different roles
- Pseudonymous attestations
- Minimal on-chain correlation
- Privacy-preserving proofs

### Attestation Privacy

**Public Information:**
- Attestation exists
- Attester identity
- Schema type
- Timestamp

**Private Information:**
- Attestation data (unless revealed)
- Specific claims
- Supporting evidence

**Usage:**
```typescript
// Create private attestation
const attestation = await eas.attest({
  schema: EMPLOYMENT_SCHEMA,
  data: encryptData({
    employee: userAddress,
    salary: 100000,
    title: "Senior Engineer"
  }, recipientPublicKey)
});

// Only recipient can decrypt
const decrypted = await decryptData(attestation.data, recipientPrivateKey);
```

## Privacy Best Practices

### For Individuals

1. **Use Separate Identities** - Don't link personal and professional
2. **Minimize On-Chain Data** - Share attestations, not raw data
3. **Control Access** - Revoke when no longer needed
4. **Encrypt Everything** - Default to encryption

### For Organizations

1. **Never Store PII On-Chain** - Use attestations instead
2. **Implement Access Controls** - Role-based permissions
3. **Audit Access Logs** - Monitor who accesses what
4. **Plan for Revocation** - Support key rotation
