# Document Identity Layer

## Overview

The foundational layer establishes unique, verifiable identifiers for real-world documents using content-addressable hashing and immutable on-chain registration.

## Core Components

### 1. Document Hashing

Documents are hashed using SHA-256 + keccak256:

```typescript
const documentBuffer = fs.readFileSync('contract.pdf');
const sha256Hash = crypto.createHash('sha256').update(documentBuffer).digest();
const integraHash = ethers.utils.keccak256(sha256Hash);
```

### 2. On-Chain Registration

Hash is registered on IntegraLedger smart contract:

```solidity
struct Document {
    bool exists;
    address owner;
    uint256 timestamp;
    string metadataURI;
    address verifier;
}

mapping(bytes32 => Document) public documents;
```

### 3. Off-Chain Storage

Actual document stored on IPFS/Arweave:

```typescript
const cid = await ipfs.add(encryptedDocument);
const metadataURI = `ipfs://${cid}`;
```

### 4. Namespace Precedents

Documents organized by namespaces for discoverability:

```
/companies/acme-corp/contracts/2024/employment-agreements/
```
